Below is a short essay for different data structures, queue, stack, set, and deque.

The queue data structure works like a queue, what is at the front gets read first, and first to be dequeued. It does not range from smallest to biggest, only first in queue to last in the queue. The stack data structure works like queue, but the top gets removed, or "pop"ed out first. A set can work like a binary search tree, "bigger or higher?". No duplicates. It can check if an element is present. It can be an organized set or not. (order number wise) A deque or "Double Ended Queue" is a type of queue that allows input on a singular side by choice, then is allowed to take from either end in the queue. These data structures can be checked if empty or full.
